# Accessibility Checklist

## Required: All Interactive Components

### Icon-only buttons
- `aria-label` or `aria-labelledby` is **required**
- Screen readers cannot infer purpose from an icon alone

### Div-based interactive elements
When using `<div>` instead of `<button>`:
```tsx
<div
  role="button"
  tabIndex={0}
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleClick();
    }
  }}
>
```

### Focus indicator
- Use `outline` on `:focus-visible` (not `:focus`)
- Recommended: `outline: 2px solid var(--effect-alpha-brand-selection)`
- Never remove focus outline without replacement

### Disabled state
- Set `disabled` attribute on native `<button>`/`<input>`
- Set `aria-disabled="true"` for visual disabled states
- Disabled must block both pointer events AND keyboard interaction
- Use `opacity.disabled` (0.5) + `cursor: not-allowed`

## Required: Specific Components

| Component | Attributes |
|-----------|-----------|
| Accordion | `aria-expanded`, `aria-controls`, `role="region"` on panel |
| Checkbox | `role="checkbox"`, `aria-checked` |
| Radio | `role="radio"`, `aria-checked`, `role="radiogroup"` on container |
| Switch | `role="switch"`, `aria-checked` |
| Select | `aria-expanded`, `aria-haspopup="listbox"` |
| Card (clickable) | `role="button"`, `tabIndex={0}`, keyboard handler |
| Table | `role="table"` (or native `<table>`), proper `<th scope>` |

## Keyboard Navigation
| Key | Action |
|-----|--------|
| `Enter` / `Space` | Activate buttons, toggles, checkboxes |
| `Arrow Up/Down` | Navigate radio groups, select options |
| `Escape` | Close dropdowns, modals |
| `Tab` | Move focus between interactive elements |

## Rules
- Never attach `onClick` to non-interactive elements without keyboard equivalent
- All form inputs must have associated `<label>` or `aria-label`
- Color must not be the only indicator of state (add icon or text)
- Touch targets: minimum 36px (sm), 40px (md), 44px (lg)
